# Architecture Decision Records (ADR)

> 记录项目关键决策。每条决策注明背景、选择、理由、影响。

---

## ADR-001 交易品种范围：主流 5 币

- 日期：2026-06-26
- 决策：交易 BTC / ETH / SOL / BNB / XRP 五个 OKX USDT 永续合约品种
- 背景：品种过多会增加数据下载时间、回测成本与过拟合风险；品种过少无法验证多品种策略鲁棒性
- 理由：5 币覆盖主流大盘币，流动性充足，资金费率数据完整，适合中低频策略验证
- 影响：数据下载范围、配置模板 `pair_whitelist`、风控单币敞口上限

## ADR-002 策略主时间框架：15m

- 日期：2026-06-26
- 决策：策略主 K 线周期为 15 分钟
- 背景：5m 手续费敏感、信号过密；1h/4h 信号过少、验证周期长
- 理由：15m 属中频，信号密度适中，回测样本充足，符合文档"中低频策略"定位
- 影响：策略 `timeframe`、数据下载粒度、回测速度、手续费占比

## ADR-003 持仓模式：双向持仓，最高 5x

- 日期：2026-06-26
- 决策：采用双向持仓模式（OKX `posMode=long_short_mode`），支持同币种多空对冲，最高杠杆 5x
- 背景：单向持仓不支持对冲，强平模型更简单但灵活性低；双向持仓便于多空策略同时运行
- 理由：5x 杠杆适中，强平距离可控；双向持仓支持 long/short 信号并存，与 FundingRateTrendStrategy 的多空逻辑契合
- 影响：强平价格估算模型（需考虑对冲仓位的保证金占用）、RISK-03 杠杆上限=5、OKX 下单参数 `posSide`

## ADR-004 小资金实盘规模：≤ 1000 USDT

- 日期：2026-06-26
- 决策：小资金实盘阶段总本金不超过 1000 USDT
- 背景：实盘初期以验证系统稳定性为主，非追求收益
- 理由：1000 USDT 足以覆盖 5 币种小仓位验证，即使全部亏损也可承受
- 影响：
  - `max_open_trades` 上限
  - 单笔下单名义价值上限（RISK-01）默认 200 USDT
  - 单币敞口上限（RISK-02）默认 300 USDT
  - 总敞口上限（RISK-03）默认 1000 USDT
  - 5x 杠杆下保证金占用约 200 USDT 即可达 1000 USDT 名义敞口

## ADR-005 OKX 账户模式：统一账户

- 日期：2026-06-26
- 决策：OKX 使用统一账户模式（Unified Account）
- 背景：统一账户下多币种共享保证金，资金利用率高；经典账户逐币种隔离
- 理由：5 币种策略在统一账户下可共享 USDT 保证金，避免资金分散；统一账户 API 更简洁
- 影响：账户余额查询接口、保证金计算、强平模型（统一账户按全仓估算）

## ADR-006 资金费结算频率假设：每 8h

- 日期：2026-06-26
- 决策：回测与实盘 PnL 统计中，资金费按每 8 小时结算一次（00:00 / 08:00 / 16:00 UTC）
- 背景：OKX 永续合约资金费标准结算频率为 8h（部分品种异常时调整为 4h/2h/1h）
- 理由：常态按 8h 估算，异常高频结算在报告中标红提示
- 影响：T2.1 资金费计算模块的结算周期参数、回测成本估算精度

## ADR-007 历史数据回测范围：至少 1 年

- 日期：2026-06-26
- 决策：回测历史数据至少覆盖 1 年（建议 2 年），含牛熊不同行情
- 背景：数据过短无法覆盖不同市场状态，walk-forward 验证无意义
- 理由：1 年为最低门槛，2 年更稳健；数据下载在 T0.2 / T1.x 完成
- 影响：数据下载时间范围参数、回测耗时
